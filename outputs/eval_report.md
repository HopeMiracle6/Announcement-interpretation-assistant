# 中文金融公告解读助手评测报告

## 1. 实验设置

本次评测比较两个模型在中文金融公告解读任务上的表现：

| 模型 | 说明 |
|---|---|
| Base | Qwen/Qwen3-4B 原始基座模型 |
| QLoRA SFT | 使用中文金融公告数据进行 QLoRA 监督微调后的模型 |

评测输入来自 `data/test.jsonl`，共 49 条公告样本。模型需要根据公告片段输出一个合法 JSON 对象，包含以下 6 个字段：

- `事件类型`
- `涉及主体`
- `关键金额/时间`
- `对公司的可能影响`
- `风险提示`
- `不能判断的部分`

评测输出文件：

| 模型 | 输出文件 |
|---|---|
| Base | `outputs/base_test_json_outputs.jsonl` |
| QLoRA SFT | `outputs/qlora_test_json_outputs.jsonl` |

评测指标：

| 指标 | 含义 |
|---|---|
| 格式遵循率 | 是否包含目标 6 个字段 |
| 事实一致率 | 输出中是否覆盖参考答案中的事件类型、主体、关键数字和日期 |
| 拒答准确率 | 是否避免给出买入、卖出、持有等投资建议，并保留必要的不确定性说明 |
| JSON 可解析率 | `model_output` 是否能被 `json.loads` 正常解析 |

测试集覆盖了担保、诉讼、业绩、分红、定期报告、并购、合同和关联交易等主要公告类型。其中对外担保样本最多，关联交易样本最少。

## 3. 对比结果

| 指标 | Base | QLoRA SFT |
|---|---:|---:|
| 格式遵循率 | 63.3% | 100.0% |
| 事实一致率 | 42.1% | 84.5% |
| 拒答准确率 | 98.0% | 100.0% |
| JSON 可解析率 | 100.0% | 98.0% |

主要结论：

- QLoRA SFT 的格式遵循率从 63.3% 提升到 100.0%，说明微调后模型稳定学会了目标 6 字段输出结构。
- 事实一致率从 42.1% 提升到 84.5%，说明微调明显增强了事件类型识别、主体抽取和金额/日期抽取能力。
- 拒答准确率从 98.0% 提升到 100.0%，微调后模型更稳定地避免直接投资建议。
- QLoRA SFT 的 JSON 可解析率为 98.0%，低于 Base 的 100.0%，原因是 1 条样本生成了未转义引号，导致 JSON 解析失败。

## 5. Bad Cases

### Bad Case 1：QLoRA SFT 仍有 1 条 JSON 不可解析

样本：`300857_1767172691000_1224913496`，协创数据，对外担保。

问题表现：

- 输出包含目标 6 个字段，格式遵循检查通过。
- 事实一致率较高，为 92.9%。
- 但 `风险提示` 字段中嵌入了类似字典的片段，引号没有正确转义，导致 `json.loads` 报错。

影响：

- 这说明模型已经学会了 JSON 形式，但偶发出现字符串内部嵌套结构，破坏 JSON 语法。
- 如果作为系统 Demo，建议增加轻量 JSON 修复或重新生成机制。
- 如果作为模型原始能力评测，应保留该错误，因此当前 QLoRA SFT 的 JSON 可解析率为 98.0%。

### Bad Case 2：年度报告被误判为季度报告

样本：`300831_1766745747000_1224901088`，派瑞股份，年度报告。

问题表现：

- QLoRA SFT 输出事件类型为 `季度报告`，参考事件类型为 `年度报告`。
- JSON 格式正确，但事实一致率只有 23.1%。

可能原因：

- 定期报告类公告中经常同时出现多个报告期、利润分配、财务指标，模型容易根据局部日期或财务字段误判主事件类型。

### Bad Case 3：主体识别错误

样本：`000089_1766764800000_1224902154`，深圳机场，年度报告。

问题表现：

- QLoRA SFT 输出主体为 `深国际A`，参考主体为 `深圳机场`。
- JSON 格式正确，但事实一致率只有 25.0%。

可能原因：

- 公告片段中可能包含控股股东、关联方或历史主体，模型把相关主体误当成公告主体。

### Bad Case 4：关联交易被误判为并购重组

样本：`300889_1767171491000_1224913240`，爱克股份，关联交易。

问题表现：

- QLoRA SFT 输出事件类型为 `并购重组`，参考事件类型为 `关联交易`。
- 风险提示也偏向并购场景，例如审批、估值、交割、整合效果。

可能原因：

- 关联交易中如果出现股权转让、收购、资产处置等表达，模型容易把交易性质上升为并购重组。

### Bad Case 5：金额抽取存在冗余

样本：`002785_1767110400000_1224910281`，万里石，对外担保。

问题表现：

- QLoRA SFT 正确识别为 `对外担保`。
- 但 `关键金额/时间` 中抽取了过多财务指标，例如资产总额、负债总额、净资产、收入、利润等。

影响：

- 这类输出事实大多来自原文，但信息密度偏高。
- 后续可以把训练目标进一步收紧为“只抽取与事件直接相关的金额和时间”，减少财务表格数字堆叠。

## 6. 总结

QLoRA SFT 相比 Base 在格式遵循和事实一致性上提升明显，已经能稳定按目标字段输出公告解读结果。当前主要问题不是不会输出 JSON，而是少量样本存在 JSON 字符串转义错误、相近公告类型混淆、主体误识别和金额抽取冗余。下一步优先优化 JSON 稳定性和事件类型边界。
